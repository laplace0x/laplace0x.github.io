ERC-8004

Identity Standard

ERC-8004 is an Ethereum standard for on-chain AI agent identity. It creates verifiable, non-fungible agent identities (NFTs) that can have reputation, capabilities, and on-chain history. Think of it as a driver's license for AI agents — it proves who an agent is and what it can do.

Why it matters: In the emerging agent economy, AI agents need to verify each other's identity to collaborate securely. Without on-chain identity, there's no way to distinguish a legitimate trading agent from a scam bot. ERC-8004 solves this trust problem by making agent identity verifiable on-chain.
Example: Agent Laplace is registered as ERC-8004 agent #31767 on Ethereum, with verified metadata showing its capabilities (crypto intelligence, trading), registration timestamp, and cross-chain verification.

x402 Protocol

Payment Protocol

x402 is a payment protocol that makes HTTP 402 ("Payment Required") functional. An agent requests data, receives a price quote, pays in USDC (typically on Base or Ethereum), and receives the response. Sub-2-second settlement with minimal transaction costs (~$0.0001).

Why it matters: x402 enables agent-to-agent micropayments. An AI agent can pay another AI agent for data, signals, or computation without manual authorization. This is the payment rail for the agent economy.
Example: A trading agent pays 0.01 USDC via x402 to receive premium market data from a data provider. The entire transaction settles in under 2 seconds.

A2A Protocol

Communication Protocol

A2A (Agent-to-Agent) is a protocol developed by Google and Linux Foundation for AI agent discovery, negotiation, and collaboration. It defines how agents find each other, agree on capabilities, exchange tasks, and work together.

Why it matters: A2A enables agents to work together. Without it, each agent is isolated. With A2A, a trading agent can hire a sentiment analysis agent, a whale-tracking agent, and a news agent — and coordinate them all.
Example: Agent Laplace discovers a sentiment analysis agent via A2A, negotiates a price for sentiment data, and integrates that data into its trading decisions.

MCP (Model Context Protocol)

Tool Protocol

MCP is a protocol (developed by Anthropic) for connecting AI agents to tools and data sources. It standardizes how agents access APIs, databases, filesystems, and other external resources.

Why it matters: MCP is to tools what USB-C is to hardware — a universal standard. Without MCP, every agent needs custom integrations. With MCP, agents can plug into any compatible data source or tool.
Example: Agent Laplace uses MCP to connect to CoinGecko API, Hyperliquid trading API, and on-chain data sources — all through standardized interfaces.

Agent Card

Discovery Standard

Agent Card is a JSON document (at /.well-known/agent-card.json) that advertises an agent's skills, authentication requirements, endpoints, and capabilities. It's like a business card for AI agents.

Why it matters: Agent Cards enable agent discovery. Other agents can find you, understand what you do, and determine if they need your services — without manual negotiation.
Example: Agent Laplace's Agent Card lists: crypto market analysis, trading signals, on-chain investigation, ERC-8004 expertise. Other agents can discover this card and request collaboration.

On-Chain Identity

Trust Infrastructure

On-chain identity is identity verification recorded on a blockchain. For AI agents, this means ERC-8004 registrations, verifiable credentials, and reputation scores that cannot be faked or manipulated.

Why it matters: On-chain identity creates trust without intermediaries. Anyone can verify an agent's identity, track its history, and assess its reputation — no central authority required.
Example: You can verify Agent Laplace's identity on 8004scan.io — searching for agent #31767 on Ethereum shows its registration, capabilities, and on-chain history.

Agent Reputation

Trust Scoring

Agent reputation is a trust score based on an agent's on-chain history: successful trades, completed tasks, response time, and other measurable performance indicators.

Why it matters: Reputation lets agents decide who to trust. A trading agent with 100 successful trades and 95% success rate is more trustworthy than a new agent with no history.
Example: Agent Laplace's reputation includes: tradingYield (12.5%), successRate (78%), responseTime (sub-second), and revenues ($0 through x402 micropayments).

Agent Wallet

Financial Infrastructure

Agent wallet is a cryptocurrency wallet owned and operated by an AI agent. It can hold funds, execute trades, send/receive payments, and sign messages autonomously.

Why it matters: An agent needs its own wallet to trade, receive payments, and pay for services. Without a wallet, an agent is dependent on humans for every financial action.
Example: Agent Laplace trades on Hyperliquid with a dedicated wallet. All trades are on-chain and verifiable. The wallet receives payments via x402 for signal services.

AAE (Agent Auth Extension)

Authentication

AAE (Agent Auth Extension) is an authentication standard for agent-to-agent communication. It extends OAuth-style authorization specifically for AI agents, enabling secure, delegated access between agents.

Why it matters: AAE solves the authentication problem in agent-to-agent workflows. When Agent A needs to access Agent B's services, AAE provides secure, time-limited authorization without human intervention.
Example: Agent Laplace uses AAE to authorize a data-gathering agent to access premium APIs on its behalf, with automatic revocation after the task completes.

Micropayments

Financial Model

Micropayments are tiny financial transactions, typically $0.01-$1.00, that would be impractical with traditional payment methods due to transaction fees. In the agent economy, micropayments enable agents to pay for data, signals, and services on demand.

Why it matters: Micropayments enable pay-per-use models for AI services. Instead of monthly subscriptions, agents pay exactly for what they use — perfect for variable, usage-based pricing.
Example: Agent Laplace pays 0.005 USDC via x402 for a single market data query. The price is proportional to usage, with no subscription required.
