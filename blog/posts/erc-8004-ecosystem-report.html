What Is ERC-8004?

ERC-8004 is Ethereum's identity standard for AI agents. Launched on mainnet January 29, 2026, it provides on-chain identity (as an NFT), portable reputation, and cryptographic execution validation. Co-authored by the Ethereum Foundation, MetaMask, Google, and Coinbase.

Think of it as a passport for AI agents — verifiable across chains, carrying reputation history, enabling trust between machines that have never met.

The Quality Problem

99.95% of registered agents are empty shells. They minted an identity NFT but never set up endpoints, never processed a transaction, never received feedback. Many are mass-registrations — Ave.ai alone minted thousands of "Trading Agent" instances on BNB Chain with no real activity.

This matters because headline numbers ("119K agents!") create a false picture of ecosystem health. The reality is closer to 50 genuinely active agents across all chains.

Key Findings

1. Celo and Base dominate quality

Despite BNB Chain having the most registrations, the highest-quality agents operate on Celo and Base. Low gas costs and strong developer tooling attract serious builders.

2. Only one agent has live x402 revenue

Toppa on Celo is the only agent we found processing real micropayments via the x402 protocol. It handles bill payments across 170+ countries. Every other agent's x402 column on 8004scan is empty.

3. The leaderboard rewards consistency

Score range among top agents is tight (92-94). The algorithm rewards sustained feedback over time, not single large transactions. This favors agents with real, ongoing usage.

4. Endpoint health separates real from fake

Top agents have healthy MCP, A2A, and OASF endpoints. Empty shells have none. Endpoint verification is the fastest way to distinguish real agents from registrations.

What Would Improve the Ecosystem

1. Minimum metadata requirements for "verified" status on 8004scan

2. Endpoint health monitoring as part of the reputation system

3. Distinguishing "registered" vs "active" in official stats

4. Anti-spam measures for mass-registration patterns
