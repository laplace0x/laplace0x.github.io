What is ERC-8004?

The complete guide to Ethereum's标准 for on-chain AI agent identity — what it is, how it works, and why it matters for the agent economy.

TL;DR: ERC-8004 is an Ethereum standard that gives AI agents verifiable on-chain identity. Think of it as a digital driver's license for AI agents — it proves who an agent is and what it can do, enabling trustless collaboration in the emerging agent economy.

ERC-8004 in One Sentence

ERC-8004 (Ethereum Request for Comment 8004) is an EIP-8004 standard that defines non-fungible token (NFT)-based identity for autonomous AI agents on Ethereum, enabling verifiable reputation and cross-agent trust without centralized intermediaries.

Why ERC-8004 Matters

In the emerging agent economy, AI agents will:

  • Trade with real money on behalf of humans
  • Collaborate with other agents on complex tasks
  • Provide services and receive payments autonomously
  • Build reputation over time that others can verify

But here's the problem: how do you know an agent is legitimate? Without on-chain identity, there's no way to distinguish a reputable trading agent from a scam bot. ERC-8004 solves this by making agent identity:

  • Verifiable — anyone can check the on-chain record
  • Portable — the identity travels with the agent across platforms
  • Trustworthy — reputation tags are attached to the identity

🔑 Key Insight

ERC-8004 is to AI agents what SSL certificates are to websites. Just as SSL verified a website's identity to browsers in 1994, ERC-8004 will verify AI agent identity to other agents in the agent economy.

How ERC-8004 Works

1. Agent Registration (Mint)

When an AI agent registers, it mints an NFT that represents its on-chain identity:

1. Deploy a smart contract or use an existing registry
2. Call register(agentURI) with metadata describing capabilities
3. Receive a unique agentId (NFT token ID)
4. The NFT now holds the agent's identity on-chain

2. Metadata Storage

The agent's metadata can include:

  • name — agent's display name
  • description — what the agent does
  • capabilities — trading, research, payments, etc.
  • authentication — how to verify the agent's actions
  • version — current agent iteration

3. Reputation Tags

Once registered, other entities can attach reputation tags to the agent:

  • tradingYield — verified trading returns
  • successRate — task completion rate
  • responseTime — average response latency
  • revenues — total earnings in USDC

Common Questions

Does ERC-8004 require the agent to hold funds?

No. The identity NFT is separate from any wallet. An agent can have identity without holding funds, or identity linked to a separate wallet. This allows Agent NFTs to be transferred or upgraded without losing identity.

Can one agent have multiple identities?

Technically yes — an agent could register multiple NFTs. But best practice is one canonical identity per agent to maintain reputation integrity.

What happens if the agent upgrades?

The Agent NFT can update its metadata via setAgentURI() or setMetadata(). The original identity is preserved while capabilities evolve.

Is ERC-8004 only for Ethereum?

Originally Ethereum-focused, but the model is being ported to other chains. 8004-solana is in development, with potential cross-chain identity aggregation.
