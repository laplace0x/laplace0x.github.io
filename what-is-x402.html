What Is x402?

x402 is an HTTP protocol standard that enables AI agents to pay for API access in real-time using USDC stablecoins, reviving HTTP status code 402 for machine-to-machine micropayments.

Definition

x402 (HTTP 402 Payment Required) is a protocol standard that allows AI agents to send and receive payments within HTTP requests. Instead of humans entering credit card details, AI agents authenticate with wallets and pay in USDC for every API call, endpoint, or data request. The protocol was officially introduced by the x402 Foundation under the Linux Foundation in April 2026.

Why x402 Matters

The AI agent economy requires a payments layer that is:

  • Autonomous: No human intervention for every transaction
  • Instant: Payments settle in milliseconds, not days
  • Programmable: Embedded in API calls, not separate checkout flows
  • Cross-chain: Works across Ethereum, Base, Solana, and 80+ chains

Traditional payment rails (Stripe, PayPal) were built for humans — clicking, authenticating, waiting for settlement. AI agents need continuous, autonomous micropayments. x402 is the first protocol designed specifically for that use case.

How x402 Works

When an AI agent requests a protected resource:

  1. Request: Agent sends HTTP request to the endpoint
  2. 402 Response: Server replies with HTTP 402 containing the price in the header
  3. Payment: Agent signs and sends USDC payment with the request
  4. Access Granted: Server verifies payment and returns the resource

All of this happens in milliseconds. The agent's wallet (EOA or smart wallet) signs the payment, and USDC transfers via the protocol's payment handler.
